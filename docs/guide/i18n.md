# Internationalization (i18n)

argparse-c provides built-in support for translating help text and error messages into different languages.

## Setting the Locale

```c
#include <argparse-c/argparse.h>

// Set locale to Spanish
argparse_set_locale("es");

// Get current locale
const char *locale = argparse_get_locale();
```

## Registering Translations

```c
// Register Spanish translations
argparse_i18n_set("es", ARGPARSE_MSG_USAGE, "Uso");
argparse_i18n_set("es", ARGPARSE_MSG_OPTIONS, "Opciones");
argparse_i18n_set("es", ARGPARSE_MSG_POSITIONAL, "Argumentos posicionales");
argparse_i18n_set("es", ARGPARSE_MSG_ERROR, "error");
argparse_i18n_set("es", ARGPARSE_MSG_REQUIRED, "requerido");
argparse_i18n_set("es", ARGPARSE_MSG_INVALID, "inválido");
argparse_i18n_set("es", ARGPARSE_MSG_MISSING, "faltante");
argparse_i18n_set("es", ARGPARSE_MSG_UNEXPECTED, "inesperado");
argparse_i18n_set("es", ARGPARSE_MSG_CONFLICT, "opciones conflictivas");

// Now set the locale
argparse_set_locale("es");

// Help output will use Spanish strings
```

## Available Message Keys

| Key | Default (English) | Description |
|-----|-------------------|-------------|
| `ARGPARSE_MSG_USAGE` | "usage" | Usage line prefix |
| `ARGPARSE_MSG_OPTIONS` | "options" | Options section title |
| `ARGPARSE_MSG_POSITIONAL` | "positional arguments" | Positional section title |
| `ARGPARSE_MSG_COMMANDS` | "commands" | Commands section title |
| `ARGPARSE_MSG_HELP` | "help" | Help option text |
| `ARGPARSE_MSG_VERSION` | "version" | Version option text |
| `ARGPARSE_MSG_ERROR` | "error" | Error prefix |
| `ARGPARSE_MSG_REQUIRED` | "required" | Required marker |
| `ARGPARSE_MSG_INVALID` | "invalid" | Invalid value message |
| `ARGPARSE_MSG_MISSING` | "missing" | Missing argument message |
| `ARGPARSE_MSG_UNEXPECTED` | "unexpected" | Unexpected argument message |
| `ARGPARSE_MSG_CONFLICT` | "conflicting options" | Mutex conflict message |

## Lookup Order

When `argparse_i18n_get(key)` is called:

1. Check current locale for a translation
2. Fall back to English ("en") translation
3. Return the key itself as a last resort

## API Reference

| Function | Description |
|----------|-------------|
| `argparse_set_locale(locale)` | Set the active locale |
| `argparse_get_locale()` | Get the current locale string |
| `argparse_i18n_get(key)` | Get translated string for a key |
| `argparse_i18n_set(locale, key, translation)` | Register a translation |

## Example: Full Spanish CLI

```c
#include <argparse-c/argparse.h>
#include <stdio.h>

int main(int argc, const char **argv) {
    // Register Spanish translations
    argparse_i18n_set("es", ARGPARSE_MSG_USAGE, "Uso");
    argparse_i18n_set("es", ARGPARSE_MSG_OPTIONS, "Opciones");
    argparse_i18n_set("es", ARGPARSE_MSG_ERROR, "error");
    argparse_set_locale("es");

    struct argparse *parser = argparse_new("miapp", "Mi aplicación");
    argparse_add_option(parser, 'v', "verbose", ARGPARSE_NARGS_0,
                        ARGPARSE_TYPE_NONE, "Salida detallada", NULL);

    struct argparse_result *result = argparse_parse(parser, argc, argv);
    // Help output now uses "Uso:" and "Opciones" in Spanish
    argparse_result_free(result);
    argparse_free(parser);
    return 0;
}
```
